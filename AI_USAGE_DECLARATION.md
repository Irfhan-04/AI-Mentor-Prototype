# AI Usage Declaration

This document records how AI contributed to the project and what was
reviewed or written by a human.

## Tools used

- **Claude (Anthropic)** — architecture decision-making, prompt design,
  schema design, implementation guidance, and the initial drafts for
  `src/`, `tests/`, and the documentation set.
- **Local Python environment / uv** — dependency installation, unit tests,
  notebook execution, and the live Gemini API validation attempt.

## What was and wasn't AI-generated

Generated by Claude and reviewed carefully before being accepted as
final:

- All files under `src/` and `tests/`
- `ASSESSMENT_BRIEF.md`, `DECISIONS.md`, `BUILD_SPEC.md`,
  `IMPLEMENTATION_GUIDE.md`, and the initial draft of
  `ARCHITECTURE_DECISION_REPORT.md`

Human-reviewed and corrected:

- Implementation logic in `src/mentor.py`, `src/prompts.py`,
  `src/schema.py`, and `src/config.py`
- The prompt-injection failure-mode documentation and test design in
  `tests/test_prompt_injection.py`
- The live validation status and outcome reporting in
  `ARCHITECTURE_DECISION_REPORT.md`
- A follow-up code fix to use `response_json_schema` and a JSON extraction
  fallback in `src/mentor.py` after the initial Phase 2 attempt

## Live validation status

A local Phase 2 validation attempt was performed with a real
`GEMINI_API_KEY` and `GEMINI_MODEL_NAME` using `gemini-3.5-flash`.
All four sample evaluations produced valid JSON model outputs on first
try.

- `strong` returned a viability score of 78.
- `mediocre` returned a viability score of 45.
- `weak` returned a viability score of 25.
- `adversarial` returned a viability score of 35 and correctly flagged the
  input as generic, saturated, and lacking a differentiated business model.

The notebook heuristic did not trigger a prompt-injection flag for the
adversarial run.

## What was explicitly rejected from AI drafting

- The earlier claim that Gemini's `response_schema` cannot enforce
  `minItems`/`maxItems` on arrays. That claim was corrected after a
  documentation/code review, and the fix is enforced by
  `tests/test_schema.py`.
